## Analysis Table

| Field name | Type | Definition |
|------------|------|------------|
| dpv_match_code | varchar(1) | Status of the Delivery Point Validation (DPV). This indicates whether or not the address is present in the USPS data.<br><br>Y — Confirmed; entire address is present in the USPS data. (To be certain the address is actually deliverable, verify that the dpv_vacant field has a value of N. You may also want to verify that the dpv_no_stat field has a value of N. However, the USPS is often several months behind in updating this data point, so only rely on the dpv_no_stat data if you are fully aware of its weaknesses and limitations.)<br>(e.g., 1600 Amphitheatre Pkwy Mountain View, CA)<br>N — Not confirmed; address is not present in the USPS data.<br>S — Confirmed by ignoring secondary info; the main address is present in the USPS data, but the submitted secondary information (apartment, suite, etc.) was not recognized.<br>(e.g., 62 Ea Darden Dr Apt 298 Anniston, AL)<br>D — Confirmed but missing secondary info; the main address is present in the USPS data, but it is missing secondary information (apartment, suite, etc.).<br>(e.g., 122 Mast Rd Lee, NH)<br>[blank or null] — The address is not present in the USPS database. |
| dpv_footnotes | varchar(32) | Information related to the delivery point validation of this address. All these footnotes have a length of 2 characters, and there may be up to 14 footnotes.<br><br>AA — Street name, city, state, and ZIP are all valid.<br>(e.g., 2335 S State St Ste 300 Provo UT)<br>A1 — Address not present in USPS data.<br>(e.g., 3214 N University Ave New York NY)<br>BB — Entire address is valid.<br>(e.g., 2335 S State St Ste 300 Provo UT)<br>CC — The submitted secondary information (apartment, suite, etc.) was not recognized. Secondary number is NOT REQUIRED for delivery.<br>(e.g., 3331 Erie Ave Apt 2 Cincinnati OH 45208)<br>C1 — The submitted secondary information (apartment, suite, etc.) was not recognized. Secondary number IS REQUIRED for delivery.<br>(e.g., 2335 S State St Ste 500 Provo UT)<br>F1 — Military or diplomatic address<br>(e.g., Unit 2050 Box 4190 APO AP 96278)<br>G1 — General delivery address<br>(e.g., General Delivery Provo UT 84601)<br>M1 — Primary number (e.g., house number) is missing.<br>(e.g., N University Ave Provo UT)<br>M3 — Primary number (e.g., house number) is invalid.<br>(e.g., 16 N University Ave Provo UT)<br>N1 — Address is missing secondary information (apartment, suite, etc.) which IS REQUIRED for delivery..<br>(e.g., 2335 S State St Provo UT)<br>PB — PO Box street style address.<br>(e.g., 555 S B B King Blvd Unit 1 Memphis TN 38103)<br>P1 — PO, RR, or HC box number is missing.<br>(e.g., Dept 126 Denver CO 802910126)<br>P3 — PO, RR, or HC box number is invalid.<br>(e.g., PO BOX 60780 FAIRBANKS AK 99706)<br>RR — Confirmed address with private mailbox (PMB) info.<br>(e.g., 3214 N University Ave #409 Provo UT)<br>R1 — Confirmed address without private mailbox (PMB) info.<br>(e.g., 3214 N University Ave Provo UT)<br>R7 — Confirmed as a valid address that doesn't currently receive US Postal Service street delivery.<br>(e.g., 6D Cruz Bay St John VI 00830)<br>TA — Primary number was matched by dropping trailing alpha.<br>(e.g., 4-C PENINSULA CTR RANCHO PALOS VERDES CA 90274)<br>U1 — Address has a "unique" ZIP Code.<br>(e.g., 100 North Happy Street 12345)<br><br>Here are some common combinations:<br><br>AABB - ZIP, state, city, street name, and primary number match.<br>AABBCC - ZIP, state, city, street name, and primary number match, but secondary does not. A secondary is not required for delivery.<br>AAC1 - ZIP, state, city, street name, and primary number match, but secondary does not. A secondary is required for delivery.<br>AAM1 - ZIP, state, city, and street name match, but the primary number is missing.<br>AAM3 - ZIP, state, city, and street name match, but the primary number is invalid.<br>AAN1 - ZIP, state, city, street name, and primary number match, but there is secondary information such as apartment or suite that would be helpful.<br>AABBR1 - ZIP, state, city, street name, and primary number match. Address confirmed without private mailbox (PMB) info. |
| dpv_cmra | varchar(1) | Indicates whether the address is associated with a Commercial Mail Receiving Agency (CMRA), also known as a private mailbox (PMB) operator. A CMRA is a business through which USPS mail may be sent or received, for example the UPS Store and Mailboxes Etc.<br><br>Y — Address is associated with a valid CMRA.<br>N — Address is not associated with a valid CMRA.<br>[blank] — Address was not submitted for CMRA verification. |
| dpv_vacant | varchar(1) | Indicates that a delivery point was active in the past but is currently vacant (in most cases, unoccupied over 90 days) and is not receiving deliveries. This status is often obtained when mail receptacles aren't being emptied and are filling up, so mail is held at the post office for a certain number of days before the delivery point is marked vacant.<br><br>Y — Address is vacant.<br>N — Address is not vacant.<br>[blank] — Address was not submitted for vacancy verification. |
| dpv_no_stat | varchar(1) | Indicates that a delivery point is listed as "no-stat" by the USPS. Technically, that means the USPS is temporarily declaring the address undeliverable. In practice, however, the USPS is often several months behind in removing addresses from the "no-stat" list, so only rely on this data point if you are fully aware of its weaknesses and limitations.<br><br>Y — USPS lists the address as "no-stat."<br>N — USPS does not list the address as "no-stat."<br>[blank] — Address was not submitted for "no-stat" verification. |
| active | varchar(1) | The API still returns this field, but in practical terms, it is deprecated. This field will contain a value of Y for every address submitted. |
| footnotes | varchar(24) | Indicates which changes were made to the input address. Footnotes are delimited by a # character. See the footnotes table below for details. |
| lacslink_code | varchar(2) | The reason for the LACSLink indication that was given (below)<br><br>A — Match: Address provided. LACSLink record match was found, and a converted address was provided.<br>00 — No Match. No converted address. No soup for you!<br>09 — Match: No new address. LACSLink matched an input address to an old address which is a "high-rise default" address; no new address was provided.<br>14 — Match: No conversion. Found a LACSLink record, but couldn't convert the data to a deliverable address.<br>92 — Match: Dropped secondary number. LACSLink record was matched after dropping the secondary number from input.<br>[blank] — No LACSLink lookup attempted. |
| lacslink_indicator | varchar(1) | Indicates whether there is an address match in the LACSLink database.<br><br>Y — LACS record match; a new address could be furnished because the input record matched a record in the master file.<br>S — LACS record - secondary number dropped; the record is a ZIP+4 street level or high-rise match. The input record matched a master file record, but the input address had a secondary number and the master file record did not.<br>N — No match; a new address could not be furnished; the input record could not be matched to a record in the master file.<br>F — False positive; a false positive record was detected.<br>[blank] — No LACSLink lookup attempted. |
| suitelink_match | varchar(5) | Indicates a match (or not) to the USPS SuiteLink data. SuiteLink attempts to provide secondary information such as "suite" or "apartment" whenever there is a match based on address and company name.<br><br>true — There was a SuiteLink match and the result is provided.<br>(e.g.,Columbia, 3700 Cabela's Blvd Lehi UT 84043)<br>false — There was no SuiteLink match. |
| enhanced_match | varchar(64) | When an address is submitted with the match parameter set to "enhanced," this field will contain additional information about the result. Multiple values may be present, separated by commas. Additional values will be added from time to time. The current possible values are:<br><br>none — No address match was found.<br>non-postal-match — A match was found within additional, non-postal address data.<br>postal-match — A match was found within postal address data.<br>missing-secondary — The address should have a secondary (e.g., apartment), but none was found in the input.<br>unknown-secondary — The provided secondary information did not match a known secondary within the address data.<br>ignored-input — The provided input contained information that was not used for a match. |
